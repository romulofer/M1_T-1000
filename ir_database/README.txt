M1 IR Remote Database
=====================

Copy the contents of this directory to the SD card at 0:/IR/

The M1 will browse these directories and .ir files from:
  Infrared > Universal Remote

Universal Remotes panel
-----------------------
The Universal/ directory is generated by tools/build_universal_ir.py, which
merges the per-brand files above into one file per device category. Within a
file every record is renamed to its canonical function (Power, Vol_Up,
Vol_Down, Ch_Up, Ch_Down, Mute, Source, Menu) and duplicate codes are dropped.

On the M1 (Infrared > Universal Remotes) you pick a category, then press a
single function icon; the M1 brute-forces THAT function across every brand in
the file until the appliance reacts (BACK aborts). Copy Universal/ to the SD
card at 0:/IR/Universal/ along with the rest of this directory.

To add a category: drop per-brand .ir files under ir_database/<Category>/, add
a CATEGORIES entry in tools/build_universal_ir.py, and re-run it -- no firmware
change needed.

File Format
-----------
All files use the Flipper Zero .ir format (compatible with Flipper IRDB).
You can add your own .ir files or download more from the Flipper IRDB
community database.

Every shipped .ir file is checked with tools/host_test/validate_ir (valid
header + at least one parseable signal). The codes in the newer categories
(Bluray, Monitor, LEDs, Streaming) are hand-authored from published brand
tables and are format-valid; on-device transmit accuracy is bench-verified
per device.

The M1 can also learn new remotes using the IR receiver.
Learned remotes are saved to 0:/IR/Learned/
